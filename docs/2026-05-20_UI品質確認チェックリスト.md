# UI品質確認チェックリスト

## 目的

UI/UX改修PRでは、機能テストだけでなく、現場作業で迷わないこと、誤操作しにくいこと、画面崩れがないことを確認する。

## 共通確認

- `python -m pytest`
- `python -m compileall pdf_splitter_tool tests`
- `python -m pdf_splitter_tool --smoke`
- 小PDFでStep 1からStep 4まで通す
- 実PDF `006_18.pdf` でStep 2のページ移動と候補処理中の操作継続性を確認する
- スクリーンショットを `artifacts/ui-screenshots/` に保存し、文字の重なり、余白、ボタン状態を確認する

## Step別確認

- Step 1: PDF一覧にファイル名、ページ数、保存場所が表示される
- Step 2: ページ状態バッジ、候補のみ表示、分割位置、OCR本文タブが確認できる
- Step 3: セグメント表で箱No、バインダーNo、連番、出力名、状態が一覧確認できる
- Step 4: 出力先、出力予定件数、要修正件数、重複回避方針がチェックリストとして確認できる
- プリセット管理: 組み込み保護、コピー作成、変数挿入、保存エラー表示が確認できる

## 合格基準

- 操作不能になる画面がない
- 出力不可の理由が画面内で分かる
- 大容量PDFでもページ移動とキャンセル操作が止まらない
- 生成物や一時PDFはGit管理対象に含めない
