## 1. 目的

本書は、PDF分割命名ツールの各画面が持つ機能、表示項目、利用者操作、入力チェック、画面遷移条件を画面単位で整理する。

全体仕様書は業務要件と復元方針を中心に扱う。本書は、実際の画面確認、改修、受入確認、社内説明で参照しやすいように、Step 1からStep 4およびプリセット管理画面の機能を独立して定義する。

## 2. 全体UX

本ツールは、複数ページPDFを業務上のまとまりごとに分割し、箱No、バインダーNo、連番などのメタデータから規則的なファイル名を作って出力するためのウィザード型GUIである。

基本フロー:

1. Step 1: PDF選択
2. Step 2: 分割
3. Step 3: 入力
4. Step 4: 出力

UX上の重点:

- 作業工程を4画面に分け、利用者が現在どの工程にいるかを把握できるようにする。
- 分割位置、命名情報、出力予定名、出力可否を出力前に確認できるようにする。
- 未入力、命名エラー、重複、既存ファイル衝突を出力前に止める。
- 大量ページPDFでも、ページ移動、候補検出中止、出力進捗確認ができるようにする。
- 事前OCR済みPDFまたはテキスト層ありPDFでは、OCR検索、検索語ハイライト、OCR本文表示、選択OCR転記を利用できるようにする。

## 3. 共通画面要素

### 3.1 ヘッダー

表示内容:

- アプリ名
- 選択中PDF名
- 選択中プリセット名
- PDF選択件数、総ページ数、分割数、入力状態、出力状態のバッジ

目的:

- 利用者が、対象PDF、プリセット、全体の作業状態を画面上部で確認できるようにする。
- Stepを移動しても、入力不足や出力可能状態を把握できるようにする。

### 3.2 Stepタブ

表示タブ:

- `1 PDF選択`
- `2 分割`
- `3 入力`
- `4 出力`

基本動作:

- タブ選択で各Stepへ移動する。
- Step間でPDF一覧、分割セグメント、メタデータ、出力チェック結果を共有する。
- Step 2以降は、Step 1でPDFが選択されていることを前提にする。

### 3.3 画面下部ステータス

表示内容:

- 出力先
- 作業フォルダ
- ショートカットの補足

目的:

- 出力先の誤認を防ぐ。
- 現場作業で使う主要ショートカットを常時確認できるようにする。

## 4. Step 1: PDF選択

### 4.1 画面目的

処理対象PDF、出力先、案件プリセット、共通項目を準備する画面である。ここで追加したPDFが以降の分割、入力、出力の対象になる。

### 4.2 主な表示項目

- セクション見出し: `PDFを選択`
- PDF選択ボタン群
- 出力フォルダ選択ボタン
- プリセット選択コンボボックス
- プリセット管理ボタン
- 共通項目入力欄
- 読込状態、読込中断ボタン
- PDF一覧
- 状態保存ボタン
- 最終保存状態

### 4.3 主な操作

| 操作 | 内容 |
| --- | --- |
| PDFを個別に選択 | 複数PDFをファイル選択で追加する。 |
| 入力フォルダを選択 | 指定フォルダ配下のPDFをまとめて追加する。 |
| 置換読込(PDF) | 現在のPDF一覧をクリアし、選択PDFで置き換える。 |
| 置換読込(フォルダ) | 現在のPDF一覧をクリアし、選択フォルダ配下のPDFで置き換える。 |
| 出力フォルダ | 分割PDFの保存先を指定する。 |
| プリセット選択 | 命名テンプレート、入力項目、しきい値を案件単位で切り替える。 |
| プリセット管理 | プリセット管理画面を開く。 |
| 共通項目を全PDFへ適用 | Step 1で入力した下書き共通値を、作成済みセグメントへ反映する。 |
| ファイル名で自動ソート | PDF読込時または一覧更新時にファイル名順へ並べる。 |
| 選択解除 | 選択中PDFを一覧から外す。 |
| 全クリア | PDF一覧を空にする。 |
| 状態を保存 | 現在の作業状態を `_pdf_split_state.json` に保存する。 |

### 4.4 入力・検証

次Stepへ進む前に満たすべき条件:

- PDFが1件以上選択されている。
- 選択PDFを読み込めている。
- 出力フォルダが指定されている。
- 必須共通項目の不足が画面上で分かる。

注意:

- Step 1の共通項目は下書きである。分割セグメントへ反映するには `共通項目を全PDFへ適用` を実行する。
- 分割セグメントが未作成の場合、共通項目の全PDF適用は実行できない。

### 4.5 出力

この画面の操作結果として、以下がアプリ状態へ反映される。

- 入力PDF一覧
- 各PDFのページ数、保存場所
- 出力フォルダ
- 選択中プリセット
- 共通メタデータ下書き

## 5. Step 2: 分割

### 5.1 画面目的

PDFをプレビューしながら、分割境界を決定する画面である。手動分割、定型分割、白紙候補、OCR検索、インデックス候補を使ってセグメントを作る。

### 5.2 主な表示項目

- セクション見出し: `分割位置を決める`
- 左ペイン: ページ状態一覧
- 中央ペイン: PDFプレビュー
- 中央上部: 現在ページ、前ページ、次ページ、現在ページ前分割
- ページ指定入力
- ズーム操作、幅に合わせる、全体表示
- 右ペイン: 現在ページ状態
- 右ペイン: 候補検出
- 右ペインタブ: 分割位置、候補、OCR本文

### 5.3 ページ状態一覧

表示内容:

- ページ番号
- 検索ヒット、索引候補、白紙候補、分割前などの状態
- 候補のみ表示切替

目的:

- ページ単位で、どこが分割候補かを俯瞰できるようにする。
- 検索、インデックス候補、白紙検出の結果を同じ一覧で確認できるようにする。

### 5.4 PDFプレビュー

機能:

- 現在ページを画像として表示する。
- 検索ヒットがある場合、プレビュー上にハイライトを表示する。
- クリックでプレビュー領域へフォーカスを移し、ショートカット操作を受け付ける。
- ズームスライダー、幅に合わせる、全体表示に対応する。

入力条件:

- PDFが選択されていること。
- OCR検索やハイライトは、事前OCR済みPDFまたはテキスト層ありPDFを入力条件とする。

### 5.5 分割位置タブ

主な操作:

| 操作 | 内容 |
| --- | --- |
| 現在ページの前に分割 | 現在表示ページを新しいセグメントの先頭として分割境界を追加する。 |
| 選択した分割を削除 | 分割位置一覧で選択した境界を削除する。 |
| 最後の分割を取り消す | 直近追加した分割境界を取り消す。 |
| Undo / Redo | 分割状態の変更を戻す、またはやり直す。 |
| 1ページごとに分割 | PDFを1ページ単位のセグメントへ自動分割する。 |
| 参照元フォルダを開く | 現在PDFの保存フォルダを開く。 |
| このファイルを改名 | 現在PDFのファイル名を確認ダイアログ付きで変更する。 |

ページ番号ルール:

- UI表示は1始まりとする。
- 内部処理は0始まりへ変換する。
- `現在ページの前に分割` は、現在ページを次セグメントの開始ページにする。

### 5.6 候補タブ

機能:

- 検索、インデックス候補、白紙検出の候補ページを一覧表示する。
- 候補ページを選択すると対象ページへ移動する。
- `候補ページの前に分割` で候補を分割境界へ反映する。

注意:

- 白紙検出は候補表示であり、自動確定ではない。
- 白紙候補は、利用者が対象ページをプレビューで確認してから分割またはスキップ判断する。

### 5.7 OCR本文タブ

機能:

- 現在ページのOCR本文またはテキスト層を表示する。
- 選択したOCR文字列を、指定したメタデータ項目へ転記する。

入力条件:

- OCR本文表示と転記は、事前OCR済みPDFまたはテキスト層ありPDFを前提とする。
- テキスト層がない画像PDFでも、通常の分割、命名、出力は継続できる。

### 5.8 ショートカット

Step 2で有効な主なショートカット:

| キー | 操作 |
| --- | --- |
| Space | 現在ページの前に分割を追加する。 |
| 左右キー | 前後ページへ移動する。 |
| Alt+左右キー | 前後のPDFへ移動する。 |
| Ctrl+Z / Ctrl+Y | Undo / Redo。 |
| Delete | 選択中の分割を削除する。 |

制約:

- 入力欄、検索欄、OCR本文にフォーカスがある場合は、文字入力やカーソル操作を優先する。
- ショートカットが通常入力を奪わないことを必須条件とする。

### 5.9 出力

この画面の操作結果として、以下が作成または更新される。

- 分割セグメント
- 分割境界一覧
- 候補ページ一覧
- OCR検索結果
- ページ状態
- 元PDFリネーム後の参照再マップ

## 6. Step 3: 入力

### 6.1 画面目的

Step 2で作成した分割セグメントごとに、出力ファイル名へ使うメタデータを入力する画面である。入力結果から予定ファイル名を生成し、出力可能状態を確認する。

### 6.2 主な表示項目

- セクション見出し: `出力名に使う情報を入力`
- セグメント一覧表
- 一括入力エリア
- 連番開始、増分
- 入力補助候補一覧
- 選択セグメント詳細
- 出力名プレビュー

### 6.3 セグメント一覧表

列:

- `#`
- `ページ`
- `箱No`
- `バインダー`
- `連番`
- `出力名`
- `状態`

目的:

- 分割範囲と命名情報を同じ表で確認する。
- 各セグメントが出力可能か、要入力かを一覧で判別する。

### 6.4 一括入力

主な操作:

| 操作 | 内容 |
| --- | --- |
| 共通値を全件へ反映 | 入力した共通値を全セグメントへ反映する。 |
| 連番を再採番 | 連番開始と増分に基づいて全セグメントの連番を振り直す。 |
| 前行メタデータコピー | 選択中セグメントへ直前行のメタデータをコピーする。 |
| 次の要修正へ移動 | 未入力または命名エラーがある次のセグメントを選択する。 |
| 入力補助候補を更新 | OCR本文などから入力候補を再生成する。 |
| 選択候補をコピー | 候補一覧で選択した文字列をクリップボードへコピーする。 |
| 出力予定フォルダを開く | 出力先フォルダを開く。 |

### 6.5 選択セグメント入力

機能:

- 選択中セグメントのメタデータを個別編集する。
- プリセットで定義された入力項目を表示する。
- 必須項目と任意項目を区別する。
- 入力変更に応じて出力名プレビューを更新する。

既定プリセットの主な項目:

- `box_no`: 箱No、必須
- `binder_no`: バインダーNo、必須
- `seq`: 連番、必須
- `company`: 会社名、任意
- `doc`: 書類名、任意

吉田エルシス既定テンプレート:

```text
{box_no:0>2}_{binder_no:0>2}_{seq:0>3}.pdf
```

出力例:

```text
01_02_003.pdf
```

### 6.6 入力・検証

出力可能になる条件:

- 必須メタデータが入力されている。
- 命名テンプレートに必要な変数が揃っている。
- 生成予定ファイル名が `.pdf` で終わる。
- ファイル名に使用できない文字が含まれていない。
- 同一出力先での重複や既存ファイル衝突が、Step 4で処理可能である。

### 6.7 出力

この画面の操作結果として、以下が更新される。

- セグメント別メタデータ
- 生成予定ファイル名
- 入力状態
- 出力前チェックで使う検証対象

## 7. Step 4: 出力

### 7.1 画面目的

出力前チェックを実行し、予定ファイル、未入力、命名エラー、重複、既存ファイル衝突、保存先を確認したうえで、一括出力する画面である。

### 7.2 主な表示項目

- セクション見出し: `出力前に確認`
- 出力前チェックボタン
- 出力実行ボタン
- 出力フォルダを開くボタン
- 出力状態
- 出力進捗
- 出力前チェックサマリー
- 選択行の処理方針
- 出力予定一覧
- 出力ログ

### 7.3 出力前チェック

確認内容:

- 出力先が指定されているか。
- 出力対象セグメントが存在するか。
- 必須メタデータが入力されているか。
- 命名テンプレートから正しいファイル名を生成できるか。
- 予定ファイル名が `.pdf` で終わるか。
- 同名ファイルや既存ファイルがあるか。
- 上書き禁止ルールが守られるか。

動作:

- エラーがある場合、`出力実行` は有効にならない。
- 警告がある場合、利用者が処理方針を確認できる。
- チェック結果は一覧とログ欄に表示する。

### 7.4 出力予定一覧

列:

- `No`
- `ページ`
- `予定ファイル名`
- `既存`
- `処理方針`
- `状態`

状態表示:

- 出力可能
- 警告
- 要修正
- 既存あり
- スキップ
- 既存再利用

### 7.5 既存ファイル衝突時の処理方針

選択肢:

| 方針 | 内容 |
| --- | --- |
| 新規名で作成 | `_2`, `_3` などの一意名で新規PDFを作成する。 |
| 既存ファイルを再利用 | 既存PDFを成果物として扱い、分割処理は行わない。SHA256をログに残す。 |
| スキップ | 該当セグメントを出力対象から外し、ログにスキップとして記録する。 |

禁止事項:

- 既存ファイルを上書きしない。

### 7.6 出力実行

動作:

- 出力前チェックでエラーがない場合のみ実行できる。
- セグメントごとに分割PDFを生成する。
- 出力進捗と現在処理状況を表示する。
- 成功、失敗、スキップ、既存再利用をログに残す。
- 出力PDFごとにSHA256を表示する。

出力物:

- 分割済みPDF
- 出力ログ
- 必要に応じた分割サマリー

### 7.7 出力不可条件

以下の場合は出力を開始しない。

- PDFが未選択。
- セグメントが未作成。
- 出力フォルダが未指定。
- 必須メタデータが不足している。
- 命名テンプレートが不正。
- 出力ファイル名が不正。
- 既存ファイル衝突の処理方針が未決定。

## 8. プリセット管理画面

### 8.1 画面目的

案件ごとの入力項目、命名テンプレート、白紙しきい値、インデックスしきい値、抽出キーワードを管理する画面である。

### 8.2 主な表示項目

- 左ペイン: プリセット一覧
- プリセットID
- 表示名
- 命名テンプレート
- 白紙しきい値
- インデックスしきい値
- テンプレート変数挿入ボタン
- 入力項目一覧
- 入力項目追加、更新、削除
- 抽出キーワード
- コピーして新規作成
- カスタム削除

### 8.3 組み込みプリセット保護

組み込みプリセットは読取専用とする。

保護対象:

- 吉田エルシス
- Legacy full metadata

制約:

- 組み込みプリセットは直接編集できない。
- 組み込みプリセットは削除できない。
- 案件別に調整する場合は `コピーして新規作成` を使う。

### 8.4 カスタムプリセット編集

編集可能項目:

- プリセットID
- 表示名
- 命名テンプレート
- 白紙しきい値
- インデックスしきい値
- 入力項目
- 抽出キーワード

入力項目の主な属性:

- 項目キー
- 表示名
- 必須
- 初期値

テンプレートで利用できる変数:

- `{box_no}`
- `{binder_no}`
- `{seq}`
- `{company}`
- `{doc}`

### 8.5 保存時チェック

保存時に確認する内容:

- プリセットIDが空でない。
- 表示名が空でない。
- 命名テンプレートが空でない。
- 入力項目キーが重複していない。
- 命名テンプレート内の変数が入力項目として定義されている。
- 組み込みプリセットを上書きしようとしていない。

保存後の動作:

- `presets.json` を更新する。
- アプリ側のプリセット選択肢を更新する。
- 選択中プリセットに応じてStep 1、Step 3、OCR転記先、出力名プレビューを再構築する。

## 9. 状態保存・復元

### 9.1 保存対象

保存対象:

- 入力PDF一覧
- 出力フォルダ
- 選択中プリセット
- PDFごとのページ数、分割範囲、メタデータ
- セグメント一覧
- 既存出力衝突時の処理方針
- 現在ページ、ズームなどのUI状態
- 白紙判定しきい値、インデックス検索しきい値などの分割支援設定

### 9.2 保存ファイル

使用ファイル:

- `_pdf_split_state.json`
- `_pdf_split_state.bak.json`
- `_pdf_split_state.tmp`

保存方針:

- 一時ファイルへ保存してから本体へ反映する。
- 既存状態ファイルはバックアップとして保持する。
- 壊れた状態ファイルがある場合はバックアップ復元を試みる。

### 9.3 操作

- `状態を保存` ボタンで保存する。
- `Ctrl+S` で保存する。
- 起動時に前回状態を復元する。

## 10. 横断ショートカット

| キー | 有効範囲 | 操作 |
| --- | --- | --- |
| Ctrl+S | 全体 | 状態保存 |
| Ctrl+Z | Step 2 | 分割操作のUndo |
| Ctrl+Y | Step 2 | 分割操作のRedo |
| Space | Step 2 | 現在ページの前に分割 |
| 左右キー | Step 2 | 前後ページへ移動 |
| Alt+左右キー | Step 2 | 前後PDFへ移動 |
| Delete | Step 2 | 選択分割を削除 |

制約:

- 入力欄、検索欄、コンボボックス、OCR本文操作中は、ショートカットより通常入力を優先する。
- ショートカットの誤発火で入力中の文字やカーソル移動を妨げない。

## 11. OCR関連機能の入力条件

OCR関連機能:

- OCR本文表示
- OCR検索
- インデックス候補検索
- 検索語ハイライト
- OCR選択テキスト転記
- Step 3入力補助候補

入力条件:

- 事前OCR済みPDF、または既存テキスト層があるPDFを使用する。
- 本ツール自体にはOCRエンジン、Tesseract、tessdata、外部OCR APIキー、`.env` を含めない。

非OCR時の扱い:

- テキスト層がない画像PDFでも、通常のPDF分割、メタデータ入力、命名、出力は利用できる。
- OCR関連機能が使えない場合は、利用者に事前OCR済みPDFの利用を案内する。
