## 1. 全体状況

ユーザー向けアプリは `apps/desktop/` の Tauri + Next.js デスクトップアプリに集約済み。PDF処理は `recovery/pdf_splitter_tool/` の Python sidecar が担当する。

| 領域 | 実装状況 | 未受入/確認事項 |
| --- | --- | --- |
| STEP1 PDF取込 | PDF複数選択、出力先指定、共通値、状態保存/復元を実装済み。 | 現場PDFでの通し確認、欠損PDF復元時の運用確認。 |
| STEP2 分割 | ページプレビュー、ページ状態一覧、分割操作、検索支援、候補表示、白紙候補表示を実装済み。 | サンプルPDF/現場PDFでの人間受入、検索支援の操作感確認。 |
| STEP3 入力 | 箱No、バインダーNo、連番、追加項目 `affix_defs`、出力名プレビュー、再採番を実装済み。 | 追加項目を含む命名の現場運用確認。 |
| STEP4 出力 | 出力前チェック、一意名採番、上書き防止、一括出力を実装済み。 | 実フォルダで既存ファイル衝突時の受入確認。 |
| 更新/配布 | Tauri updater、GitHub Releases、`latest.json` 配布ラインを実装済み。現行 `main` は v0.1.2 リリース候補。v0.1.1 は既存安定版として残す。 | 旧バージョンからの更新検出、インストール、再起動後バージョン表示のE2E確認。 |

## 2. STEP別詳細

| STEP | 実装済み機能 | 未受入/手動確認 |
| --- | --- | --- |
| STEP1 | PDF追加/除外/全クリア、出力先選択/リセット、箱No/バインダーNo共通値、`_pdf_split_state.json` 保存/復元。 | PDFを複数選択し、出力先と共通値を設定してSTEP2へ進めること。状態保存後の再起動復元。 |
| STEP2 | 現在PDF単位の分割、分割点削除、全解除、ページ番号入力、ズーム、幅合わせ、全体表示、ページ状態一覧。 | 1366px前後でプレビュー、操作群、右ペインが破綻しないこと。 |
| STEP2検索 | 現在PDF固定検索、契約書用語モーダル、複数用語選択、PDFプレビュー上ハイライト、OCR本文内強調、候補検索、白紙候補。 | テキスト層ありPDFで検索、結果クリック、ハイライト、候補ページ確認が成立すること。 |
| STEP3 | 固定3項目（箱No、バインダーNo、連番）、追加項目最大2件、先頭/末尾挿入、OCR本文から追加項目への転記、プレビュー名表示。 | 追加項目あり/なしの両方で、プレビュー名と出力予定名が現場運用に合うこと。 |
| STEP4 | 出力前チェック、既存ファイル回避、`_2` / `_3` 以降の一意名採番、確定済み出力パス尊重、一括出力。 | 実出力先で同名ファイルがある場合に上書きされないこと。 |

## 3. 主要仕様の実装状態

| 仕様 | 現在の扱い |
| --- | --- |
| STEP2検索範囲 | 現在操作中PDFのみ。旧UIの `現在PDF / 全PDF` 切替は使わない。 |
| 検索用語プリセット | ブラウザ `localStorage` の `pdf-organizer.step2SearchTerms.v1` に保存する。`_pdf_split_state.json` とは分離する。 |
| STEP2 sidecar | `page_thumbnail`、`page_text`、`search_text`、`search_highlights`、`index_candidates`、`blank_candidates` を追加済み。 |
| OCR方針 | 事前OCR済みPDFまたは既存テキスト層ありPDFの検索支援は実装済み。画像PDFへのOCRエンジン内蔵は後回し。 |
| 追加項目 | `affix_defs` 最大2件。位置は `prefix` または `suffix`。空値は出力名から除外する。 |
| 命名互換 | 追加項目なしでは `{box_no:0>2}_{binder_no:0>2}_{seq:0>3}.pdf` を維持する。 |
| 状態保存 | `_pdf_split_state.json` に `affix_defs` を含める。検索用語プリセットは含めない。 |
| リリース | `apps/desktop/RELEASE.md` に Tauri updater、署名鍵、GitHub Releases、`latest.json` の手順を集約する。 |

## 4. 検証コマンド

リポジトリ単位の自動回帰検証:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\verify.ps1
```

desktop 側の個別確認:

```powershell
cd apps\desktop
npm run typecheck
npm run test:desktop
npm run test:step-render
```

sidecar/API境界の重点確認:

```powershell
cd recovery
python -m pytest tests/test_sidecar.py tests/test_desktop_shell_contract.py
```

サンプルPDFのsidecarスモーク:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\sample-pdf-smoke.ps1 -PdfPath "C:\path\to\sample.pdf"
```

注意:

- `scripts/verify.ps1` は Python pytest、desktop 側 JS 回帰テスト、desktop typecheck をまとめて確認する。
- `scripts/verify.ps1` は Tauri実機E2E、アプリ内updater E2E、現場PDFの人間受入確認の代替ではない。
- `sample-pdf-smoke.ps1` は元PDFを変更せず、一時フォルダで `pdf_info` / preview / preflight / export / state roundtrip を確認する。
