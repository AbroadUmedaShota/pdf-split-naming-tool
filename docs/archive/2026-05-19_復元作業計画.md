# 2026-05-19 復元作業計画

## 目的

元ソースが見つからない前提で、配布済み `PDF分割命名ツール_v3_1_1` を改修可能な状態へ戻す。

## 作業順

1. バイトコードから現行仕様を整理する。
2. 画面構成、データ構造、PDF分割処理、命名処理、状態保存処理を分離して記録する。
3. `recovery/` 配下に Python プロジェクトとして復元ソースを作る。
4. 配布EXEと同等の基本動作をサンプルPDFで確認する。
5. 改修要望を受けて、復元ソース側へ変更を入れる。
6. PyInstaller one-folder 形式で再ビルドし、共有フォルダへ配布できる形にする。

## 初期の復元対象

- `MetadataField`
- `Segment`
- `PdfInfo`
- `PreviewScope`
- `PageOcrData`
- `SplitActionDecision`
- `FilenameBuildResult`
- `StateManager`
- `PdfProcessor`
- `PdfSplitterApp`

## 最初に検証する受入条件

- PDFを選択できる。
- ページプレビューが表示される。
- 手動分割を設定できる。
- 命名テンプレートから出力予定名を作れる。
- 一括出力で分割PDFが生成される。
- `_pdf_split_state.json` に作業状態を保存できる。

## 注意点

- 抽出済み `.pyc` は解析資料であり、直接編集対象にはしない。
- 元ソース相当を復元する場合も、最終的には読みやすい新規 `.py` として整理する。
- 共有フォルダの既存配布物は上書きせず、新しいバージョン名で出力する。
