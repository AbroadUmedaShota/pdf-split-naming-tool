# app.py責務分割設計メモ

## 目的

`recovery/pdf_splitter_tool/app.py` は、2026-05-20時点で約2,000行あり、Step 1〜4の画面構築、PDF描画、OCR検索、状態保存、出力、プリセット管理が1ファイルに集中している。

実PDF受入確認と配布準備は完了したため、今後の改修では挙動を変えずに小さく責務分割し、UI変更時の回帰リスクを下げる。

## 現在の責務

| 責務 | 主な位置 | 内容 |
| --- | --- | --- |
| アプリ起動・全体状態 | `PdfSplitterApp.__init__` | Tk root、プリセット、PDF一覧、セグメント、状態変数の初期化 |
| 状態保存/復元 | `build_state_payload`、`save_state`、`load_state_on_startup` | `_pdf_split_state.json` の保存と復元 |
| Step 1 PDF選択 | `_build_step1`、`add_pdf_paths`、`refresh_pdf_list` | PDF追加、フォルダ読込、共通項目下書き、一覧更新 |
| Step 2 分割/OCR | `_build_step2`、`render_current_page_async`、`start_text_search`、`start_blank_scan` | ページ表示、分割位置、検索、白紙候補、OCR本文 |
| Step 3 メタデータ | `_build_step3`、`refresh_segment_list`、`resequence_segment_metadata` | 一括入力、連番再採番、セグメント別メタデータ編集 |
| Step 4 出力 | `_build_step4`、`refresh_output_summary`、`run_output` | 出力前チェック、分割PDF出力、SHA256ログ |
| プリセット管理 | `PresetManagerDialog` | プリセット一覧、コピー作成、入力項目編集、保存 |

## 分割方針

- 先に業務ロジックを増やさず、UIクラスの移動だけで済む単位から始める。
- `PdfSplitterApp` が保持する状態を急に分散させない。
- 既存テスト、`python -m pdf_splitter_tool --smoke`、`python scripts/gui_smoke.py` を必ず通す。
- 1PRにつき1責務だけを移動し、機能追加やUI文言変更を混ぜない。

## 推奨モジュール構成

| 追加候補 | 役割 | 優先度 |
| --- | --- | --- |
| `preset_manager_dialog.py` | `PresetManagerDialog` を移動する | 高 |
| `ui_theme.py` | 色、ttk style、Listbox/Textの共通スタイルを移動する | 中 |
| `step2_controller.py` | ページ状態、候補一覧、分割位置の派生計算を移動する | 中 |
| `output_controller.py` | 出力前チェックと出力ログ生成を移動する | 中 |
| `state_adapter.py` | Tk変数と保存JSONの変換を移動する | 低 |

## 最初の安全な分割単位

最初は `PresetManagerDialog` を `recovery/pdf_splitter_tool/preset_manager_dialog.py` へ移動する。

理由:

- Step 1〜4の通常業務フローから独立している。
- 既に1つのクラスとしてまとまっており、切り出し境界が明確。
- `PdfSplitterApp` 側の変更は `from .preset_manager_dialog import PresetManagerDialog` と `open_preset_manager` の呼び出し維持に限定できる。
- `python scripts/gui_smoke.py` でプリセット管理画面の起動とスクリーンショット取得を自動確認できる。

## 初回分割PRの受入条件

- `PresetManagerDialog` 移動以外のUI文言・業務ロジックを変更しない。
- `python -m pytest` が成功する。
- `python -m compileall pdf_splitter_tool tests scripts` が成功する。
- `python -m pdf_splitter_tool --smoke` が成功する。
- `python scripts/gui_smoke.py` が成功し、プリセット管理スクリーンショットが保存される。
- 差分レビューで `PdfSplitterApp` のStep 1〜4処理に意味的な変更がないことを確認する。

## 後回しにすること

- Step 2の描画・検索・白紙検出の分割は、非同期処理とUI状態が絡むため初回には扱わない。
- 状態保存/復元の分割は、JSON互換性確認が必要なため初回には扱わない。
- `PdfSplitterApp` の状態オブジェクト化は、既存GUIテストを増やしてから検討する。
