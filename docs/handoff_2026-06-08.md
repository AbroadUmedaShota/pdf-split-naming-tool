### 計測根拠（らでんの「測ってから」を実行）

spawn が描画の約64倍＝**プロセス再利用が支配的勝ち筋**。60枚サムネ概算 7.6s→1.0s(warm後0.47s)。

### Phase A (#86): Python `--serve`
- `__main__.py` に `--serve`、`sidecar.py` に `serve(in,out)` + `compact_response()`。
- JSONLines(1行1req/1resp)・毎応答 flush・例外でループ不死・`sys.stdout`→stderr 保護。
- ワンショット(`--sidecar-request`)は無改変・parityテストでバイト等価担保。回帰ゼロ。

### Phase B (#87): Rust 常駐デーモン
- `SidecarState(Mutex<Option<SidecarDaemon>>)` を Tauri State 管理。Mutex でリクエスト直列化。
- stdout を専用 reader スレッドで drain→mpsc。per-request timeout = `recv_timeout`。
- timeout/死亡で「全破棄→次回 lazy respawn」。broken pipe のみ安全再送(送信前死亡=未配送)。
- **Disconnect 時は再送しない**（export 等の二重実行防止・レビュー Blocker 対応）。
- 2段 timeout（heavy=base×4）。終了 kill 3重（Drop + RunEvent::Exit try_lock + Python stdin-EOF）。
- escape hatch: `PDF_ORGANIZER_SIDECAR_ONESHOT=1` で旧経路(`run_sidecar_oneshot`)へ即退避。
- retry/respawn を `run_sidecar_exchange` に切出し spawn 注入でテスト化。
- 3観点レビュー(品質/構造/性能)実施→ ab8cfd2 で Blocker/Major/Medium/Low を修正済み。
