# PDF分割命名ツール 最小構成仕様

作成日: 2026-05-30  
位置づけ: 現段階で優先するMVP仕様。既存仕様書に含まれるOCR、白紙検出、プリセット管理などの高度機能は、本書では後回しの拡張候補として扱う。

## 1. 目的

PDFを取り込み、ページ単位で分割範囲を決め、箱No・バインダーNo・連番で命名して、上書きせずにPDF出力する。

現段階では、元アプリの全機能再現よりも、ヨシダエルシス向けの基本運用を迷わず実行できることを優先する。

## 1.1 実装構成

ユーザーが操作するアプリは `apps/desktop/` の Tauri + Next.js デスクトップアプリ1つに統一する。

`recovery/pdf_splitter_tool/` はPython sidecarとして扱う。PDFページ数取得、プレビュー生成、分割、命名、出力前チェック、状態保存を担当する処理部品であり、独立した2つ目のGUIアプリではない。

## 1.2 画面幅方針

本アプリは、ローカルPDFを選択し、PDFプレビュー、分割一覧、入力表、出力確認を見ながら作業するデスクトップ業務ツールである。そのため、実運用の主対象はデスクトップPCおよび小型ノートPC幅とする。

対象範囲:

- 主対象: 1024px以上のPC/ノートPC幅。
- 重点確認: 1366px前後の一般的な業務PC幅。
- 最低限確認: 小型ノートPC幅で主要操作が画面外へ破綻しないこと。
- 任意確認: スマホ相当幅で致命的な重なりや横方向のページ全体崩れがないこと。

非対象:

- スマートフォンでPDF分割・命名・出力の実作業を完結できること。
- モバイル専用の操作導線、表レイアウト、PDFプレビュー最適化。

モバイル幅対応は、画面が壊れないための保険として扱う。MVPの設計・検証・改善工数は、PC幅での作業効率を優先する。

## 2. 最小構成の対象範囲

### 残す機能

- PDFファイルを選択する。
- 出力フォルダを選択する。
- PDFページをプレビューする。
- 前ページ、次ページへ移動する。
- 現在ページの前で分割する。
- 1ページごとに分割する。
- 最後の分割を取り消す。
- 分割セグメント一覧を表示する。
- セグメントごとに箱No、バインダーNo、連番を入力する。
- 連番を再採番する。
- 出力ファイル名をプレビューする。
- 出力前チェックを行う。
- 一括出力する。
- 既存ファイルがある場合は `_2`, `_3` などで自動的に一意名を作る。
- `_pdf_split_state.json` に作業状態を保存する。

### 後回しにする機能

- ページ回転
- ページ除外

- 現在ページだけ抽出
- 元PDFファイル名変更
- プリセット管理

- 既存ファイルの「再利用」「スキップ」選択
- 詳細機能タブ

## 3. 画面構成

最小構成では、画面は4つに絞る。

1. PDF取込
2. 分割
3. 入力
4. 出力

## 4. 画面別仕様

### 4.1 PDF取込

目的:

- 処理対象PDFと出力先を指定する。

表示項目:

- PDF選択ボタン
- 選択済みPDF一覧
- 出力フォルダ選択ボタン
- 共通入力欄: 箱No、バインダーNo
- 状態保存ボタン

操作:

- PDFを追加する。
- 選択PDFを一覧から外す。
- 出力フォルダを指定する。
- 箱No、バインダーNoを入力する。
- 状態を保存する。

次へ進む条件:

- PDFが1件以上選択されている。
- 出力フォルダが指定されている。

### 4.2 分割

目的:

- PDFを見ながら分割位置を決める。

表示項目:

- PDFプレビュー
- 現在ページ番号
- 前ページ、次ページボタン
- 現在ページの前で分割ボタン
- 1ページごとに分割ボタン
- 最後の分割を取り消すボタン
- 分割一覧

操作:

- 前後ページへ移動する。
- 現在ページの前に分割境界を追加する。
- PDFを1ページ単位で分割する。
- 最後に追加した分割境界を取り消す。

ページ番号ルール:

- 画面表示は1始まり。
- 内部処理はPDFライブラリへ渡す直前に0始まりへ変換する。
- 「現在ページの前で分割」は、現在ページを次セグメントの開始ページにする。

### 4.3 入力

目的:

- 分割セグメントごとに命名に必要な情報を入力する。

表示項目:

- セグメント一覧
- ページ範囲
- 箱No
- バインダーNo
- 連番
- 出力ファイル名プレビュー
- 連番再採番ボタン

操作:

- セグメントを選択する。
- 箱No、バインダーNo、連番を入力する。
- 連番を再採番する。
- 出力予定名を確認する。

必須項目:

- 箱No
- バインダーNo
- 連番

### 4.4 出力

目的:

- 出力前チェックを行い、問題がなければ分割PDFを一括出力する。

表示項目:

- 出力前チェックボタン
- 出力実行ボタン
- 出力予定一覧
- チェック結果
- 出力ログ

チェック内容:

- 出力フォルダが指定されている。
- 出力対象セグメントが存在する。
- 箱No、バインダーNo、連番が入力されている。
- 生成ファイル名が `.pdf` で終わる。
- Windowsファイル名として使えない文字が正規化される。
- 既存ファイルがある場合、上書きせず一意名で出力できる。

出力動作:

- セグメントごとにPDFを生成する。
- 既存ファイルは上書きしない。
- 同名ファイルがある場合は `_2`, `_3` などを付けて出力する。

## 5. 命名仕様

現段階では命名テンプレートを固定する。

```text
{box_no:0>2}_{binder_no:0>2}_{seq:0>3}.pdf
```

出力例:

```text
01_02_003.pdf
```

入力値は文字列として扱い、テンプレート適用時にゼロ埋めする。

## 6. 状態保存

保存対象:

- 入力PDF一覧
- 出力フォルダ
- 分割セグメント
- セグメントごとの箱No、バインダーNo、連番
- 現在ページ

保存ファイル:

- `_pdf_split_state.json`
- `_pdf_split_state.bak.json`
- `_pdf_split_state.tmp`

保存方針:

- 一時ファイルへ書き出してから本体へ置き換える。
- 既存状態ファイルはバックアップへ退避する。
- 保存失敗時に作業状態を壊さないことを優先する。

## 7. 最小受入条件

1. PDFを選択できる。
2. 出力フォルダを指定できる。
3. PDFページをプレビューできる。
4. 現在ページの前で分割できる。
5. 1ページごとに分割できる。
6. 分割セグメント一覧を確認できる。
7. 箱No、バインダーNo、連番を入力できる。
8. 連番を再採番できる。
9. `01_02_003.pdf` 形式の出力名を作れる。
10. 未入力がある場合は出力前に止められる。
11. 既存ファイルを上書きせず、一意名で出力できる。
12. 一括出力で分割PDFを生成できる。
13. 作業状態を保存し、再起動後に復元できる。
14. 1024px以上のPC/ノートPC幅で主要操作が成立する。
15. スマホ相当幅では、実作業最適化ではなく、致命的な表示崩れがないことを確認する。

## 8. 後続拡張の扱い

後回し機能は、最小構成が現場運用で成立することを確認してから、別Issueまたは別仕様として追加判断する。

優先順位の目安:

1. 現場作業で手戻りが多いもの
2. 入力ミスや出力ミスを減らすもの
3. 大量PDF処理の時間を大きく短縮するもの
4. 特定案件だけで必要なもの
