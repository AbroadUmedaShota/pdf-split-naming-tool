# 元アプリ解析・デグレ防止仕様

作成日: 2026-05-20
対象Issue: #29
対象元アプリ: `PDF分割命名ツール_v3_1_1.exe`

## 1. 目的

UI/UX刷新や吉田エルシス向け改修を進める前に、元アプリが持っていた業務上必要な機能、導線、安全条件を整理し、復元版でデグレを起こさないための基準を定義する。

本書では、元アプリと復元版の差分を以下に分類する。

- `復元済み`: 現在の復元版で実装済み。
- `仕様変更`: 現場FBにより意図的に変更したもの。
- `必須復元`: 元アプリ主要価値または安全性に関わるため復元が必要。
- `後回し可`: 便利機能だが初期運用を止めるほどではない。
- `要再確認`: 元アプリ内に痕跡はあるが、実機操作で詳細確認が必要。

## 3. 元アプリの業務フロー

元アプリはウィザード形式で、以下の流れを持つ。

1. Step 1: 処理対象PDF、案件プリセット、共通項目、出力先を選択する。
2. Step 2: PDFをプレビューし、手動分割、定型分割、白紙分割、OCR検索/インデックス候補検索で分割範囲を作る。
3. Step 3: 分割単位ごとにメタデータを入力し、生成予定ファイル名を確認する。
4. Step 4: 出力前チェックを行い、一括出力し、ログ・進捗・出力結果を確認する。
5. Ctrl+Sで状態保存し、次回起動時に状態復元できる。

画面下部には以下の操作ヒントがある。

```text
Ctrl+S 保存  /  Ctrl+Z,Y Undo/Redo  /  Alt+←,→ PDF移動
```

## 5. 横断要件

### 5.1 状態保存・復元

元アプリで確認された要素:

- `_pdf_split_state.json`
- `_pdf_split_state.bak.json`
- `_pdf_split_state.tmp`
- `作業状態を保存しました。`
- `前回状態を復元しました`
- `バックアップから復元`
- 旧形式状態ファイルの非対応表示

復元版には `StateManager` はあるが、GUIフローへの保存/復元統合が不足している。これはデグレ防止上の必須復元項目とする。

対応: #36

### 5.2 元PDFリネーム

元アプリには `rename_current_pdf_file`、`_remap_path_references_after_rename` の痕跡がある。復元版では未実装。
元PDF本文は変更しない前提を維持し、ファイル名変更だけを条件付きで許可する。

必須条件:

- 実行前に確認ダイアログを出す。
- 成功後、PDF一覧、分割範囲、OCR/検索キャッシュ、状態ファイル内参照を再マップする。
- 失敗時は旧パス参照を維持し、利用者に理由を表示する。

対応: #36

### 5.3 ショートカット

元アプリの画面下部表示:

- Ctrl+S: 保存
- Ctrl+Z/Y: Undo/Redo
- Alt+←/→: PDF移動

復元版ではStep 2のSpace/左右キーは実装済みだが、元アプリのショートカット互換は未達。
ショートカットは入力欄、検索欄、OCR本文操作を妨げないことを必須条件とする。

対応: #33, #36

## 6. PR #28 デグレ確認結果

PR #28 は、復元版の現行主要フローを見やすくするUI/UX改善であり、以下の点ではデグレを起こしていない。

- PDF選択から分割、メタデータ入力、出力前チェック、一括出力までの現行復元フローを維持している。
- Step 2のページ状態、候補、分割位置の視認性を改善している。
- Step 3でセグメント状態を一覧確認できるようにしている。
- Step 4で出力可否と要修正理由を確認しやすくしている。
- プリセット管理の組み込み保護を分かりやすくしている。

一方、PR #28 は元アプリ完全互換を達成するものではない。以下はPR #28のマージ前に「既知の未復元差分」として明示し、別Issueで追跡する必要がある。

- OCR検索語ハイライト: #30
- OCR選択テキスト転記: #31
- Step 1取込/共通項目/読込管理: #32
- Step 2ズーム/Undo/Redo/ページ操作: #33
- Step 3入力補助/前行コピー/未解決行ナビゲーション: #34
- Step 4出力進捗/ログ/SHA256: #35
- 状態保存/復元/元PDFリネーム再マップ: #36

## 7. マージ判断

PR #28 は、以下を満たす場合に「UI/UX改善PR」としてマージ可能と判断する。

1. PR #28が元アプリ完全互換ではないことをPR本文で明記する。
2. 本書の差分表と #30〜#36 が、未復元差分の追跡先としてリンクされている。
3. PR #28の検証結果に、元アプリ主要価値を壊していないことを記録する。
4. UI刷新後も #30〜#36 の復元実装が可能な画面構造であることを確認する。

逆に、PR #28を「元アプリ互換完了」とみなしてマージすることは禁止する。
