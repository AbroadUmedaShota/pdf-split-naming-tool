# 配布前チェックリスト

## 対象

- 復元アプリ: `recovery/`
- 配布形式: PyInstaller one-folder
- 標準出力先: `recovery/dist/<VersionName>/`
- ZIP出力先: `recovery/dist/<VersionName>.zip`

## ビルド手順

```powershell
cd recovery
python -m pip install -e ".[test,build]"
.\scripts\build_distribution.ps1 -VersionName pdf-split-naming-tool-recovery-YYYYMMDD
```

同じ `VersionName` のフォルダまたはZIPが存在する場合、上書きせずに停止する。

## 保存場所

EXE配布時は、アプリ作業フォルダをEXEと同じフォルダにする。

- `presets.json`: EXEと同じフォルダ
- `_pdf_split_state.json`: EXEと同じフォルダ
- `_pdf_split_state.bak.json`: EXEと同じフォルダ
- `_pdf_split_state.tmp`: EXEと同じフォルダ
- `output/`: EXEと同じフォルダ配下の既定出力先

`--smoke` ではGUIを開かず、上記の保存予定場所をJSONで確認できる。

```powershell
.\dist\<VersionName>\<VersionName>.exe --smoke --smoke-output .\dist\<VersionName>\smoke-result.json
```

## 配布前確認

- `python -m compileall pdf_splitter_tool tests` が成功する。
- `python -m pytest` が成功する。
- PyInstaller one-folderビルドが成功する。
- `<VersionName>.exe --smoke` が `smoke-result.json` を作成する。
- `smoke-result.json` の `work_dir` がEXEフォルダを指している。
- `presets_path` がEXEフォルダ直下の `presets.json` を指している。
- `state_path` がEXEフォルダ直下の `_pdf_split_state.json` を指している。
- EXEを起動し、GUIが表示される。
- 小さなPDFを追加し、Step 2でプレビューが表示される。
- `Split by 1 Page` でセグメントが作成される。
- Step 3で `箱No`、`バインダーNo`、`連番` を入力できる。
- 出力名が `01_02_003.pdf` 形式になる。
- Step 4で分割PDFを出力できる。
- 既存の共有フォルダ配布物を上書きしていない。

## 注意

- `recovery/dist/` と `recovery/build/` はGit管理対象外。
- 配布前の実データ検証は、個人情報や契約情報を含むPDFをGit管理対象に入れない。
- ZIPを共有フォルダへ置く前に、既存配布物とは別フォルダ名で展開確認する。
