## 1. 目的

PDFを取り込み、ページ単位で分割範囲を決め、箱No・バインダーNo・連番で命名して、上書きせずにPDF出力する。

現段階では、元アプリの全機能再現よりも、ヨシダエルシス向けの基本運用を迷わず実行できることを優先する。

## 2. 対象利用者

- 社内で紙資料やスキャン済みPDFを案件・箱・バインダー単位に整理する担当者。
- PDFを確認しながら、分割位置と出力ファイル名を手作業で確定する担当者。
- 配布前チェック、問い合わせ対応、保守を行う開発・運用担当者。

## 3. 対象環境

- 主対象: Windows上で動作するデスクトップアプリ。
- 操作対象: 1024px以上のPC/ノートPC幅。
- 重点確認: 1366px前後の一般的な業務PC幅。
- 最低限確認: 小型ノートPC幅で主要操作が画面外へ破綻しないこと。
- 任意確認: スマホ相当幅で致命的な重なりや横方向のページ全体崩れがないこと。

スマートフォンでPDF分割・命名・出力の実作業を完結できることは、MVPの受入条件に含めない。

## 4. 実装構成

ユーザーが操作するアプリは `apps/desktop/` の Tauri + Next.js デスクトップアプリ1つに統一する。

`recovery/pdf_splitter_tool/` はPython sidecarとして扱う。PDFページ数取得、プレビュー生成、分割、命名、出力前チェック、状態保存を担当する処理部品であり、独立した2つ目のGUIアプリではない。

現行の主要sidecar機能:

| command | 役割 |
| --- | --- |
| `pdf_info` | PDFのページ数、ページ番号、命名テンプレートを返す。 |
| `page_preview` | 指定ページのプレビュー画像をData URLで返す。 |
| `preflight` | 出力前チェックを行い、出力可否、出力予定パス、警告を返す。 |
| `export` | セグメントごとにPDFを分割出力する。 |
| `state_load` | 作業状態を読み込む。 |
| `state_save` | 作業状態を保存する。 |

## 5. MVP対象範囲

MVPで残す機能:

- PDFファイルを選択する。
- 出力フォルダを選択する。
- PDFページをプレビューする。
- 前ページ、次ページへ移動する。
- 現在ページの前で分割する。
- 1ページごとに分割する。
- 最後の分割を取り消す。
- 分割セグメント一覧を表示する。
- セグメントごとに箱No、バインダーNo、連番を入力する。
- 連番を再採番する。
- 出力ファイル名をプレビューする。
- 出力前チェックを行う。
- 一括出力する。
- 既存ファイルがある場合は `_2`, `_3` などで自動的に一意名を作る。
- `_pdf_split_state.json` に作業状態を保存する。

MVPで後回しにする機能:

- OCR検索、OCR本文表示、OCR選択テキスト転記。
- Step 3入力補助候補。
- 白紙検出、インデックス候補検索。
- ページ回転、ページ除外、ページ順入れ替え、現在ページだけ抽出。
- 元PDFファイル名変更。
- プリセット管理、Legacyプリセット。
- 出力履歴画面。
- 既存ファイルの「再利用」「スキップ」選択。
- 詳細機能タブ。
- 旧Tkinter GUIの復活。

後回し機能は、MVPが現場運用で成立することを確認してから、別Issueまたは別仕様として追加判断する。

## 6. 画面構成

画面は4工程に絞る。

1. PDF取込
2. 分割
3. 入力
4. 出力

画面レイアウトはB案ベースとし、左固定ナビではなく、上部の横ステッパーで4工程を表示する。各工程は、左一覧ペイン、中央作業ペイン、右側の進捗/次アクションパネルで構成する。

共通レイアウト:

- 上部: アプリ名、現在状態、4ステップの横ステッパー、PDF件数、分割件数、入力状態、出力先状態。
- 左ペイン: 各Stepの対象一覧。PDF一覧、分割セグメント、入力対象、出力予定を表示する。
- 中央ペイン: 各Stepの主操作。PDF選択、PDFプレビュー、命名入力、出力前チェックを表示する。
- 右ペイン: 完了条件、警告、次に行う主操作を表示する。

## 7. 画面別要件

### 7.1 PDF取込

目的:

- 処理対象PDFと出力先を指定する。

主な表示項目:

- PDF選択ボタン。
- 選択済みPDF一覧。
- 出力フォルダ選択ボタン。
- 共通入力欄: 箱No、バインダーNo。
- 状態保存ボタン。

主な操作:

- PDFを追加する。
- 選択PDFを一覧から外す。
- PDF一覧を全クリアする。
- 出力フォルダを指定する。
- 出力フォルダ指定をリセットする。
- 箱No、バインダーNoを入力する。
- 状態を保存する。

次へ進む条件:

- PDFが1件以上選択されている。
- 出力フォルダが指定されている。

### 7.2 分割

目的:

- PDFを見ながら分割位置を決める。

主な表示項目:

- PDFプレビュー。
- 現在ページ番号。
- 前ページ、次ページボタン。
- 現在ページの前で分割ボタン。
- 1ページごとに分割ボタン。
- 最後の分割を取り消すボタン。
- 分割一覧。

主な操作:

- 前後ページへ移動する。
- 現在ページの前に分割境界を追加する。
- PDFを1ページ単位で分割する。
- 最後に追加した分割境界を取り消す。

ページ番号ルール:

- 画面表示は1始まり。
- 内部処理はPDFライブラリへ渡す直前に0始まりへ変換する。
- 「現在ページの前で分割」は、現在ページを次セグメントの開始ページにする。

### 7.3 入力

目的:

- 分割セグメントごとに命名に必要な情報を入力する。

主な表示項目:

- セグメント一覧。
- ページ範囲。
- 箱No。
- バインダーNo。
- 連番。
- 出力ファイル名プレビュー。
- 連番再採番ボタン。

主な操作:

- セグメントを選択する。
- 箱No、バインダーNo、連番を入力する。
- 連番を再採番する。
- 出力予定名を確認する。

必須項目:

- 箱No。
- バインダーNo。
- 連番。

### 7.4 出力

目的:

- 出力前チェックを行い、問題がなければ分割PDFを一括出力する。

主な表示項目:

- 出力前チェックボタン。
- 出力実行ボタン。
- 出力予定一覧。
- チェック結果。
- 出力ログ。

チェック内容:

- 出力フォルダが指定されている。
- 出力対象セグメントが存在する。
- 箱No、バインダーNo、連番が入力されている。
- 生成ファイル名が `.pdf` で終わる。
- Windowsファイル名として使えない文字が正規化される。
- 既存ファイルがある場合、上書きせず一意名で出力できる。

出力動作:

- セグメントごとにPDFを生成する。
- 既存ファイルは上書きしない。
- 同名ファイルがある場合は `_2`, `_3` などを付けて出力する。

## 8. 命名仕様

現段階では命名テンプレートを固定する。

```text
{box_no:0>2}_{binder_no:0>2}_{seq:0>3}.pdf
```

出力例:

```text
01_02_003.pdf
```

入力値は文字列として扱い、テンプレート適用時にゼロ埋めする。

Windowsファイル名として使えない文字は `_` に正規化する。正規化後に空になる場合は出力不可または安全な既定名として扱い、実装側の出力前チェック結果に従う。

## 9. 状態保存

保存対象:

- 入力PDF一覧。
- 出力フォルダ。
- 分割セグメント。
- セグメントごとの箱No、バインダーNo、連番。
- 現在PDF。
- 現在ページ。

保存ファイル:

- `_pdf_split_state.json`
- `_pdf_split_state.bak.json`
- `_pdf_split_state.tmp`

保存方針:

- 一時ファイルへ書き出してから本体へ置き換える。
- 既存状態ファイルはバックアップへ退避する。
- 破損した状態ファイルは退避し、読み込み可能なバックアップまたは一時ファイルから復元する。
- 保存失敗時に作業状態を壊さないことを優先する。

## 10. 非機能要件

安全性:

- 既存の出力PDFを上書きしない。
- 出力前チェックで未入力、出力先未指定、出力対象なしを検出する。
- 状態保存は一時ファイルとバックアップを使い、保存失敗時の破損リスクを下げる。

操作性:

- 4工程の現在位置と完了状態を横ステッパーで把握できる。
- 左一覧、中央作業、右進捗の3ペインで、対象・操作・次アクションを分ける。
- 業務PC幅で主要操作が見切れないことを優先する。

保守性:

- ユーザー向けGUIは `apps/desktop/` に集約する。
- PDF処理はPython sidecarへ寄せ、UIと処理責務を分離する。
- 旧配布物解析や復元メモは正本要件から分離し、`docs/archive/` に保存する。

## 11. 受入条件

1. PDFを選択できる。
2. 出力フォルダを指定できる。
3. PDFページをプレビューできる。
4. 現在ページの前で分割できる。
5. 1ページごとに分割できる。
6. 分割セグメント一覧を確認できる。
7. 箱No、バインダーNo、連番を入力できる。
8. 連番を再採番できる。
9. `01_02_003.pdf` 形式の出力名を作れる。
10. 未入力がある場合は出力前に止められる。
11. 既存ファイルを上書きせず、一意名で出力できる。
12. 一括出力で分割PDFを生成できる。
13. 作業状態を保存し、再起動後に復元できる。
14. 1024px以上のPC/ノートPC幅で主要操作が成立する。
15. スマホ相当幅では、実作業最適化ではなく、致命的な表示崩れがないことを確認する。
