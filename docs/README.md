# PDF分割命名ツール ドキュメント索引

このフォルダでは、現行MVPの仕様を `01_要件定義書.md` に集約する。過去の調査、復元計画、旧仕様、チェックリストは履歴資料として `archive/` に保存する。

## 正本

- `01_要件定義書.md`: 現行MVPの要件定義書。実装・受入判断ではこの文書を優先する。

## 現役の補助文書

- `02_実装計画書.md`: 要件定義書を実装へ落とし込む計画書。STEP1-4の担当割り当て、既存実装の検証・補完タスク、要確認事項を整理する。
- `2026-05-31_デザインシステム.md`: MVP画面設計とCSS実装で守るUI方針。

## 方針・検討中

- `03_技術方針書.md`: 技術スタックと構造を既存実装に縛られず再検討した方針書。現行MVP仕様ではなく、次期構成の判断材料として扱う。
- `04_移行ロードマップ.md`: 技術方針を段階移行へ落とし込むロードマップ。フェーズ別の受入条件、Stop/Goゲート、ロールバック基準を整理する。

## 参考・履歴

- `archive/`: 初期調査、復元計画、旧アプリ解析、旧仕様、社内展開メモ、各種チェックリスト。
- `superpowers/plans/`: 過去の実装計画メモ。

## アセット

- `assets/`: 仕様書や説明資料で参照する画像などの管理対象アセット。

## 更新ルール

- 現行MVPの仕様変更は、まず `01_要件定義書.md` に反映する。
- UIの見た目やコンポーネント規則は `2026-05-31_デザインシステム.md` に反映する。
- 履歴資料は内容を書き換えず、必要に応じて `archive/` 内で参照する。
