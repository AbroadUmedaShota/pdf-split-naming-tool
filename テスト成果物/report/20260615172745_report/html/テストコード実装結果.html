3. 使用テストフレームワーク

  • Python: pytest 9.1.0（recovery/.venv/Scripts/python.exe -m pytest）。pyproject の optional-deps test = ["pytest>=8"] に宣言済みの既存テスト基盤。本フェーズで venv に pytest を導入（製品依存ではなくテストランナーの有効化）。PyMuPDF（fitz）は導入済みを利用。
  • 既存資産: node 検査スクリプト（node:assert/strict ベースの自作ハーネス。TypeScript を typescript で都度トランスパイルしてロジックを検証）、PowerShell スモークスクリプト、Rust cargo test。

実装方針: TC-CB の -k <名> はテスト設計の仕様レベル名であり、既存テストファイルには同名関数が存在しない（既存テストは同等の挙動を別名でカバー）。本フェーズでは TC-CB 1件＝独立判定1テスト関数を満たすため、実 public API を直接叩く新規テストを1ファイルに集約。テスト関数名に TC-CB-ID を埋め込み、docstring に TC/TD/TV/TA/Risk を記載してトレースを担保した。
