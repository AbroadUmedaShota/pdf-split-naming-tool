# 手動受入チェックリスト（リリース前・実機確認）

形式ばった全44ケース（`テストケース_人間実行.md`）から、リリース判断に必要な項目を優先度つきで抽出した実行用リスト。実機（Tauri デスクトップアプリ）で人が操作して確認する。

## 前提

- 起動: リリース判定ではインストール版 v0.1.4 を起動して確認する。開発中の再現確認だけ、リポジトリ直下の `start-dev.cmd` または `apps/desktop` の `npm run tauri dev` を使う。
- 用意するもの: テキスト層あり/なしの実PDF 数件（複数ページ・11ページ程度のものが望ましい）。書き込める出力先フォルダ。
- 結果欄に Pass / Fail / 備考 を記入する。Fail は再現手順とスクショを残す。
- 自動確認済みの範囲: ブラウザE2E、dev preview E2E（検索ハイライト残留なし、表示モード/ズーム/Ctrl+ホイール、STEP2/STEP3矢印ナビと入力欄フォーカス中ガード、検索支援、インデックス候補、白紙候補）、取込失敗時の画面表示・console errorなし・スクリーンショット/JSON証跡添付、複数PDF全件成功時の一覧順・件数・STEP2プレビュー反映、PDF情報取得後の初回プレビュー失敗時に取込済みとプレビュー失敗を分ける表示、出力先キャンセル/再設定/デスクトップ戻し、PDF選択中の二重起動防止、PDF追加後の出力先・共通項目維持、インストール版UI E2E（通常スモークのファイル選択ダイアログはハーネス固定。複数PDFの一部失敗時に正常PDFだけ取り込む挙動、PDF以外/存在しないパスの明示エラー、複数PDFの連続追加、重複排除、追加順、現在PDFの選択状態維持、取込後の1件除外、全クリア、再取込を含む）、インストール版状態保存/復元E2E（保存、再起動、復元、分割と入力値復元）、インストール版コア一周E2E（生成2ページPDFとローカルOCR確認PDF 2種の取込、分割、命名入力、出力前チェック、2件出力、出力後の作成済み表示、二重出力ガード、既存同名PDFがある場合の上書き防止）、インストール版ネイティブダイアログE2E（Windowsの実PDF選択ダイアログから一覧反映、実フォルダ選択ダイアログから出力先反映）、生成12ページPDFとローカルOCR確認PDF 2種の `sample-pdf-smoke.ps1` は成功済み。業務本番PDFの分割範囲・命名妥当性は本チェックリストで確認する。

## 判定

- 優先度A が全Pass = リリースの最低ライン。1つでもFailなら NoGo。
- 優先度B のFailは今セッション追加分の不具合 → 修正してから出す。
- 優先度C のFailは重大度で個別判断（多くは後続改善で可）。
