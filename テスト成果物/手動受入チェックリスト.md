# 手動受入チェックリスト（リリース前・実機確認）

形式ばった全44ケース（`テストケース_人間実行.md`）から、リリース判断に必要な項目を優先度つきで抽出した実行用リスト。実機（Tauri デスクトップアプリ）で人が操作して確認する。

## 前提

- 起動: リポジトリ直下の `start-dev.cmd` をダブルクリック（または `apps/desktop` で `npm run tauri dev`）。
- 用意するもの: テキスト層あり/なしの実PDF 数件（複数ページ・11ページ程度のものが望ましい）。書き込める出力先フォルダ。
- 結果欄に Pass / Fail / 備考 を記入する。Fail は再現手順とスクショを残す。

## 優先度A：リリース必須（コア動線・正しさ）

| # | 確認項目 | 手順 | 期待 | 結果 |
|---|---|---|---|---|
| A1 | コア一周 | STEP1でPDFを複数追加・出力先指定 → STEP2で分割 → STEP3で箱No/バインダーNo/連番入力 → STEP4で出力実行 | 一連が詰まらず完走し、出力先に分割PDFが生成される | |
| A2 | **プレビュー描画**（今回直した中核バグ） | STEP2/3で実PDFを開く | ページ画像が実際に表示される（「プレビューを取得できませんでした」が出ない）。ページ送りで別ページが表示される | |
| A3 | 出力ファイルの中身 | A1の出力PDFを実際に開く | 分割範囲が指定どおり。命名が `01_02_003.pdf` 形式（追加項目なし時）で意図どおり | |
| A4 | 既存ファイル衝突回避 | 同じ出力先に2回出力する | 既存を上書きせず `_2` `_3` で別名採番される。出力前チェックで既存が可視化される | |
| A5 | 命名: 使用不可文字・パス長 | 箱No等に `/` `:` 等を入れる／非常に深い出力先を指定 | 使用不可文字は `_` 等に正規化、パス長超過は出力前チェックでブロック・文言表示 | |
| A6 | テキスト層なしPDF | テキスト層のないPDFで分割〜出力 | 「テキスト層なし」表示が出るが、分割・命名・出力は通常どおり継続できる | |
| A7 | 状態保存/復元 | 作業途中で保存 → アプリ再起動 → 復元 | PDF構成・分割・入力値が復元される。欠損PDFがある場合の挙動が破綻しない | |
| A8 | **更新機能（updater）** | 旧版から新版への更新検出→インストール→再起動 | 更新が検出・適用され、再起動後にバージョン表示が新版になる（※要・配布物） | |

## 優先度B：今セッションで追加/変更した機能（実機未確認・要確認）

| B3 | 作成済みの可視化 | 出力後にSTEP4の一覧を見る | 出力済み行が「作成済み」（緑系）で、未出力「出力可能」と区別される | |
| B4 | 失敗分のみ再出力 | （再現できれば）一部だけ失敗する状況を作り「失敗したN件を再出力」 | 成功済みは触らず失敗分だけ出る。重複が生まれない | |
| B5 | 結果サマリの区別 | 出力結果ボックスを確認 | 成功=緑/失敗=赤、失敗時はファイル名＋理由が明細表示される | |

## 判定

- 優先度A が全Pass = リリースの最低ライン。1つでもFailなら NoGo。
- 優先度B のFailは今セッション追加分の不具合 → 修正してから出す。
- 優先度C のFailは重大度で個別判断（多くは後続改善で可）。
