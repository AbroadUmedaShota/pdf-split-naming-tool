# E2Eテスト一覧 STEP1優先

作成日: 2026-06-17

## 方針

STEP1のPDF取込は、次の2レーンで固定する。

- ブラウザE2E: Tauriのファイル選択とsidecar応答をmockし、UI状態遷移を高速に確認する。
- 実機E2E: インストール版アプリで、同梱sidecarと実PDFを使って確認する。通常の実機E2EはOSファイル選択ダイアログをテストハーネスで固定し、PDF解析とプレビューは実際のTauri command / 同梱sidecarを通す。PDF選択と出力先フォルダ選択の実ダイアログは、Windowsの実ダイアログを操作する `test:installed-native-dialog` で別途確認する。

dev preview（`?dev=import`）はサンプル状態を描画する確認用であり、PDF選択や`pdf_info`失敗を検知できないため、STEP1取込の合否判定には使わない。

## 2026-06-17 確認メモ

- ブラウザE2Eは `TC-E2E-S1-001` から `TC-E2E-S1-006`、`TC-E2E-S1-011`、`TC-E2E-S1-012`、`TC-E2E-S1-020`（PDF以外/存在しないパス）、`TC-E2E-S1-021`、`TC-E2E-S1-022`、および `TC-E2E-011` の12件が成功。
- インストール版 v0.1.4 の同梱sidecarで `--smoke`、`pdf_info`、`page_preview` が成功。
- `npm run test:installed-step1` で、インストール版アプリを起動し、パスワード付きPDFの明示エラー、PDF以外/存在しないパスの明示エラー、一覧未混入、エラー後の複数PDF再取込（一部失敗時に正常PDFだけ取込）、日本語・空白入りパスの実PDF取込、2ページ表示、複数PDFの連続追加、重複排除、追加順、現在PDFの選択状態維持、取込後の1件除外、全クリア、再取込、プレビュー表示、出力先設定、STEP2遷移まで成功。
- `npm run test:installed-core` で、インストール版アプリを起動し、生成2ページPDFのSTEP1取込、STEP2分割、STEP3命名入力、STEP4出力前チェック、出力実行、`01_01_001.pdf` / `01_01_002.pdf` の2件作成、出力後の `作成済み` 表示、出力実行ボタン無効化まで成功。
- `PDF_ORGANIZER_INSTALLED_CORE_PDF_PATH` でローカルOCR確認PDF 2種を指定して `npm run test:installed-core` を実行し、各11ページPDFのSTEP1取込、STEP2分割、STEP3命名入力、STEP4出力前チェック、出力実行、`01_01_001.pdf` / `01_01_002.pdf` の2件作成まで成功。`PDF_ORGANIZER_INSTALLED_CORE_EXPECTED_PAGES` を指定した場合はページ数も厳密確認する。
- `npm run test:installed-native-dialog` で、インストール版アプリを起動し、Windowsの実PDF選択ダイアログに生成2ページPDFを指定して一覧反映と2ページ表示まで成功。続けてWindowsの実フォルダ選択ダイアログに一時出力先を指定し、出力先反映まで成功。
- `npm run test:installed-state` で、インストール版アプリを起動し、取込、分割、2セグメント入力、状態保存、アプリ再起動、状態復元、PDF一覧、出力先、分割、入力値復元まで成功。
- `TC-E2E-S1-007` から `TC-E2E-S1-010` は実装済み。PDF選択ダイアログと出力先フォルダ選択ダイアログ自体は `TC-E2E-S1-018` で補完済み。業務本番PDFは手動受入で補完する。
- 追加スモークとして、生成した12ページ・テキスト層ありPDFとローカルOCR確認PDF 2種で `scripts/sample-pdf-smoke.ps1` を実行し、ページ数取得、先頭/最終ページプレビュー、出力前チェック、1件出力、状態保存/復元が成功。業務上の分割範囲・命名妥当性は人間受入で確認する。
