# E2Eテスト一覧 STEP1優先

作成日: 2026-06-17

## 方針

STEP1のPDF取込は、次の2レーンで固定する。

- ブラウザE2E: Tauriのファイル選択とsidecar応答をmockし、UI状態遷移を高速に確認する。
- 実機E2E: インストール版アプリで、同梱sidecarと実PDFを使って確認する。OSファイル選択ダイアログはテストハーネスで固定し、PDF解析とプレビューは実際のTauri command / 同梱sidecarを通す。

dev preview（`?dev=import`）はサンプル状態を描画する確認用であり、PDF選択や`pdf_info`失敗を検知できないため、STEP1取込の合否判定には使わない。

## 2026-06-17 確認メモ

- ブラウザE2Eは `TC-E2E-S1-001` から `TC-E2E-S1-006`、`TC-E2E-S1-011`、および `TC-E2E-011` の8件が成功。
- インストール版 v0.1.4 の同梱sidecarで `--smoke`、`pdf_info`、`page_preview` が成功。
- `npm run test:installed-step1` で、インストール版アプリを起動し、パスワード付きPDFの明示エラー、一覧未混入、エラー後の複数PDF再取込（一部失敗時に正常PDFだけ取込）、日本語・空白入りパスの実PDF取込、2ページ表示、プレビュー表示、出力先設定、STEP2遷移まで成功。
- `TC-E2E-S1-007` から `TC-E2E-S1-010` は実装済み。OSファイル選択ダイアログ自体はハーネス固定のため、手動受入で補完する。
- 追加スモークとして、生成した12ページ・テキスト層ありPDFで `scripts/sample-pdf-smoke.ps1` を実行し、ページ数取得、先頭/最終ページプレビュー、出力前チェック、1件出力、状態保存/復元が成功。現場PDFそのものではないため、現場PDF受入は別途残す。
